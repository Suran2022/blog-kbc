# 博客项目产品设计与开发方案

## 一、需求分析

### 1.1 项目概述

本项目是一个类似新闻网站的博客系统，主要面向博主发布内容和用户阅读内容的场景。系统分为前台展示和后台管理两部分，前台用户仅可浏览文章，不能注册、登录和评论，所有内容由博主通过后台管理系统发布。

### 1.2 核心需求

**前台展示系统：**
- 无菜单导航
- 搜索功能
- Logo展示
- 主内容区分为两栏布局：
  - 左侧：文章卡片列表区
  - 右侧：侧边栏（包含分类、最新、贡献三个功能区）

**后台管理系统：**
- 博主登录功能
- 文章管理（发布、编辑、删除）
- 分类管理
- 系统设置（如Logo上传等）

### 1.3 用户角色

- **访客**：只能浏览文章，使用搜索功能，查看分类、最新文章和贡献内容
- **博主**：拥有后台管理权限，可以发布、编辑、删除文章，管理分类等

## 二、产品设计报告

### 2.1 功能模块设计

#### 2.1.1 前台展示系统

**1. 首页**
- Logo区：展示博客标识
- 搜索区：提供关键词搜索功能
- 文章列表区：以卡片形式展示文章，包含标题、摘要、发布时间、分类标签等
- 侧边栏：
  - 分类区：展示所有文章分类，点击可筛选该分类下的文章
  - 最新区：展示最新发布的文章列表
  - 贡献区：展示博客贡献者或特色内容

**2. 文章详情页**
- 文章完整内容展示
- 相关文章推荐
- 返回首页链接

**3. 搜索结果页**
- 搜索关键词展示
- 符合条件的文章列表
- 搜索结果数量统计

**4. 分类筛选页**
- 当前分类名称展示
- 该分类下的文章列表
- 文章数量统计

#### 2.1.2 后台管理系统

**1. 登录页**
- 用户名/密码登录表单
- 记住登录状态选项

**2. 控制台首页**
- 数据概览：文章总数、分类数量、访问统计等
- 快捷操作入口

**3. 文章管理**
- 文章列表：展示所有文章，支持搜索、筛选、排序
- 文章编辑：富文本编辑器，支持图片上传、格式设置等
- 文章发布：设置分类、发布时间等

**4. 分类管理**
- 分类列表：展示所有分类，支持增删改查
- 分类编辑：设置分类名称、描述等

**5. 系统设置**
- 基本信息设置：博客名称、Logo上传、描述等
- 账户安全：修改密码等

### 2.2 数据结构设计

#### 2.2.1 核心数据表

**1. 用户表(User)**
- id: 主键
- username: 用户名
- password: 密码（加密存储）
- nickname: 昵称
- avatar: 头像URL
- email: 邮箱
- role: 角色（默认为admin）
- create_time: 创建时间
- update_time: 更新时间

**2. 文章表(Article)**
- id: 主键
- title: 文章标题
- content: 文章内容
- summary: 文章摘要
- category_id: 分类ID（外键）
- view_count: 浏览次数
- create_time: 创建时间
- update_time: 更新时间
- status: 状态（草稿/已发布）
- cover_image: 封面图片URL

**3. 分类表(Category)**
- id: 主键
- name: 分类名称
- description: 分类描述
- create_time: 创建时间
- update_time: 更新时间

**4. 系统设置表(Setting)**
- id: 主键
- site_name: 站点名称
- logo_url: Logo图片URL
- description: 站点描述
- keywords: 站点关键词
- footer_info: 页脚信息

**5. 贡献者表(Contributor)**
- id: 主键
- name: 贡献者名称
- description: 贡献描述
- link: 相关链接
- create_time: 创建时间

### 2.3 交互流程设计

#### 2.3.1 前台用户浏览流程

1. 用户访问博客首页
2. 浏览文章列表或通过侧边栏功能区筛选内容
3. 使用搜索功能查找特定内容
4. 点击文章卡片进入文章详情页阅读完整内容
5. 通过分类链接筛选查看特定分类的文章

#### 2.3.2 后台管理流程

1. 博主登录后台管理系统
2. 在控制台查看网站数据概览
3. 进入文章管理页面创建/编辑文章
4. 设置文章分类、封面图等信息
5. 发布文章或保存为草稿
6. 管理分类、系统设置等

## 四、技术方案

### 4.1 技术选型

#### 4.1.1 前端技术栈

- **框架**：Vue.js 3 + Vue Router + Pinia
- **UI组件库**：Element Plus
- **HTTP客户端**：Axios
- **编辑器**：TinyMCE/WangEditor（富文本编辑器）
- **构建工具**：Vite

#### 4.1.2 后端技术栈

- **框架**：Spring Boot 2.7.x
- **ORM框架**：MyBatis-Plus
- **安全框架**：Spring Security
- **数据库**：MySQL 8.x
- **缓存**：Redis（可选，用于提升性能）
- **文件存储**：本地存储/阿里云OSS（用于图片上传）

### 4.3 API设计

#### 4.3.1 前台API

**1. 文章相关**
- `GET /api/articles` - 获取文章列表（分页）
- `GET /api/articles/{id}` - 获取文章详情
- `GET /api/articles/search` - 搜索文章
- `GET /api/articles/category/{categoryId}` - 获取特定分类下的文章
- `GET /api/articles/latest` - 获取最新文章

**2. 分类相关**
- `GET /api/categories` - 获取所有分类

**3. 贡献者相关**
- `GET /api/contributors` - 获取贡献者列表

**4. 系统设置相关**
- `GET /api/settings` - 获取系统设置（如Logo等）

#### 4.3.2 后台API

**1. 认证相关**
- `POST /api/admin/login` - 管理员登录
- `POST /api/admin/logout` - 退出登录

**2. 文章管理**
- `POST /api/admin/articles` - 创建文章
- `PUT /api/admin/articles/{id}` - 更新文章
- `DELETE /api/admin/articles/{id}` - 删除文章
- `GET /api/admin/articles` - 获取文章列表（带筛选、排序）

**3. 分类管理**
- `POST /api/admin/categories` - 创建分类
- `PUT /api/admin/categories/{id}` - 更新分类
- `DELETE /api/admin/categories/{id}` - 删除分类
- `GET /api/admin/categories` - 获取分类列表

**4. 系统设置**
- `PUT /api/admin/settings` - 更新系统设置
- `POST /api/admin/upload` - 上传文件（如Logo、文章图片等）

## 六、总结

本博客项目是一个典型的内容管理系统，采用前后端分离架构，前端使用Vue.js框架，后端使用Spring Boot框架。系统分为前台展示和后台管理两部分，前台用户只能浏览文章，后台管理员可以发布和管理文章。

项目特点：
1. 简洁的两栏式布局，突出内容展示
2. 完善的搜索和分类功能，方便用户查找内容
3. 强大的后台管理功能，便于博主管理内容
4. 前后端分离架构，提高开发效率和系统性能

按照上述开发计划，预计项目总开发周期为16天左右。在开发过程中，将严格遵循用户提出的7条开发规范，确保项目高质量交付。
